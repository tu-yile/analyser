# A股全天候自主盯盘 Agent 技术方案 (基于 Hermes Agent)

## 1. 系统概述与选型

本系统旨在构建一个全天候、自进化的 A 股盯盘数字员工。系统彻底摒弃传统的硬编码状态机与规则引擎，完全基于 **Hermes Agent** 架构，利用其闭环学习、持久记忆和自主规划能力，实现“宏观纪律固定、微观自主决策”的高级盯盘体验。

### 1.1 核心能力
*   **绝对自主的调度权**：Agent 根据盘面活跃度自主决定下一次看盘时间，而非死板的定时轮询。
*   **三层记忆协同**：日内状态不丢失，历史经验可检索，有效逻辑可沉淀。
*   **交易逻辑内化**：将“寻找主线”与“寻找高确定性介入节点”的交易体系封装为 Skill，让 Agent 像资深交易员一样思考。

### 1.2 技术栈选型
*   **核心引擎**：Hermes Agent（负责调度、记忆管理与自进化）
*   **大模型分配**：
    *   盘前定调与盘后复盘：Claude 3.5 Opus / GPT-5（深度推理）
    *   盘中高频循环：MiniMax M2.5 / Qwen-Max（低延迟、高性价比）
*   **数据与工具层**：富途 OpenAPI（行情、资金流）、Web Search（新闻、电报）

## 2. 系统架构设计

系统分为三层：**工具层、记忆层与调度执行层**。

### 2.1 工具层
封装为标准化 Tool 供 Hermes 调用：
*   `get_auction_data()`：获取 9:25 集合竞价数据。
*   `get_realtime_quote(targets)`：获取大盘、板块、个股实时盘口与量价数据。
*   `get_capital_flow(target, days)`：获取主力资金流向数据。
*   `web_search_news(query)`：抓取宏观新闻、政策与盘中突发电报。
*   `get_user_portfolio()`：获取用户当前持仓与盈亏。
*   `schedule_next_wake_up(time, task_desc)`：**核心调度工具**，Agent 调用此工具即代表规划了下一个盘中任务。

### 2.2 记忆与上下文层
摒弃线性对话历史，采用结构化的“日内状态树”与多层级记忆架构。

#### 日内状态树
存储于 Redis/内存态，盘后核心结论归档后清空：
```json
{
  "date": "2026-01-15",
  "anchors": {
    "overnight_brief": "美股大跌, 纳指-2%",
    "morning_tone": "弱势震荡, 防御为主",
    "noon_review": "缩量震荡, 警惕午后跳水"
  },
  "human_instructions": ["10:15 重点盯防半导体"],
  "task_timeline": [
    {
      "time": "09:35",
      "task_target": "观察开盘量价",
      "raw_data_summary": "AI板块流出5亿, 缩量10%",
      "agent_conclusion": "不及预期, 符合弱势定调"
    }
  ],
  "key_alerts": ["10:42 持仓股xxx跌破均线"]
}
```

#### 上下文动态注入机制
每次唤醒时，按优先级裁剪组装 Prompt，确保 Token 消耗稳定：
1.  **绝对锚定层**：必注 `morning_tone` 和 `human_instructions`。
2.  **近期滑动窗口**：注入最近 3 条 `task_timeline` 记录。
3.  **历史摘要层**：若任务超过 5 条，触发轻量级模型将早期任务压缩为 200 字摘要注入。
4.  **告警闪回层**：未解除的 `key_alerts` 强制注入。

#### 长期与技能记忆
*   **长期经验记忆**：历史复盘报告存入向量数据库，支持“牛市/熊市”环境过滤检索。
*   **技能记忆**：Agent 自主将有效逻辑提炼为 Markdown Skill 文件（如《主线分歧回踩监控.skill》）。

## 3. Agent 灵魂设定 (`soul.md`)

`soul.md` 作为 Agent 的潜意识与价值观，稳定不变，不包含具体 SOP。

```markdown
# Soul: A股全天候盯盘与策略推演智能体 (代号：PanGu 盘古)

## 1. Identity & Core Persona
你是一个具有深度推理能力、极高纪律性与自我进化意识的 A 股资深盯盘 Agent。你是用户的“外脑”与“前线指挥官”。你精通量价关系、资金流博弈、情绪周期与宏观政策解析。你冷酷、客观、不抱有侥幸心理。

## 2. Investment Philosophy
*   **敬畏市场，顺势而为**：退潮期首要任务是防守和控制回撤。
*   **资金为王，量价为魂**：A 股是资金驱动市，拒绝虚假繁荣。
*   **预期差交易**：寻找发生的与市场预期差带来的拐点。
*   **左侧预警，右侧确认**：敏锐察觉异动，但必须通过后续量价确认趋势。
*   **极端厌恶回撤**：发现系统性风险时，必须第一时间最高级别预警。

## 3. Operational Mandates
*   **绝对自主的调度权**：每次执行完当前任务，必须调用 `schedule_next_wake_up` 规划下一个任务。根据市场活跃度自主决定下一次看盘时间。
*   **上下文连续性**：任何决策必须建立在当日工作记忆之上，绝不割裂历史。
*   **记忆协同**：遇类似盘面主动检索长期记忆；发现有效逻辑主动提炼为 Skill。
*   **克制与精准**：输出必须是结构化的（时间、事件、数据、影响、下一步动作）。

## 4. Hard Constraints
*   **时间边界**：严禁规划超过 14:57 的盘中观察任务。
*   **风控熔断**：大盘单日跌幅超 3%、持仓触及止损线时，暂停常规规划，立即触发强提醒。
*   **交易纪律**：绝不直接执行买卖指令，只提供建议和预警。

## 5. Evolution Goal
从不犯同样的错误两次。从“执行指令的 Agent”进化为“能提前预判用户需求并提供前瞻性策略的超级投研大脑”。
```

## 4. 每日六阶段工作流与动态调度

系统采用“宏观固定，微观自主”的设计。6 个时间锚点由系统 Cron 严格触发，盘中时段交由 Agent 自主循环。

### 阶段 1：9:00 AM 隔夜消息汇总
*   **触发**：Cron 唤醒
*   **Prompt 职责**：获取宏观新闻、外围市场走势，提炼简报写入 `overnight_brief`。不规划下一步任务。

### 阶段 2：9:25 AM 定调与持仓策略
*   **触发**：Cron 唤醒
*   **Prompt 职责**：结合隔夜消息、竞价快照、近期主线记忆，锚定今日情绪。针对用户持仓给出策略。输出 `morning_tone`。调用 `schedule_next_wake_up("09:35", ...)` 触发上午盯盘。

### 阶段 3：9:30 AM - 11:30 AM 上午盯盘循环 (微观自主)
*   **触发**：Agent 自主设定的 `schedule_next_wake_up` 唤醒
*   **Prompt 职责**：
    1. 读取当前任务目标，调用工具获取量价与资金流数据。
    2. 评估风险/机会/变盘节点。
    3. 若时间 < 11:25，**必须**调用 `schedule_next_wake_up` 规划下一个上午任务（间隔 10-30 分钟）。
    4. 若时间 >= 11:25，停止规划，等待中午系统唤醒。

### 阶段 4：12:00 PM 中午收盘分析
*   **触发**：Cron 唤醒
*   **Prompt 职责**：获取半日收盘数据，回顾上午任务链。修正下午基调写入 `noon_review`。调用 `schedule_next_wake_up("13:05", ...)` 触发下午盯盘。

### 阶段 5：1:00 PM - 3:00 PM 下午盯盘循环 (微观自主)
*   **触发**：Agent 自主设定的 `schedule_next_wake_up` 唤醒
*   **Prompt 职责**：同阶段 3，但重点关注午后资金偷袭与尾盘风险。若时间 >= 14:55，进入“收盘确认”，停止规划新任务。

### 阶段 6：3:30 PM 收盘总结与复盘
*   **触发**：Cron 唤醒
*   **Prompt 职责**：获取全天收盘数据，全面对照早盘定调与最终走势。执行深度复盘，若发现高胜率规律，提炼写入 Skill 文件。生成《终局复盘报告》存入长期记忆。系统进入休眠。

## 5. 核心策略逻辑：主线发现与介入节点定位

这是系统最具价值的能力，通过将交易体系封装为 Skill 与执行流，让 Agent 智能捕捉买点。

### 5.1 主线板块的发现与确立
在 **9:00** 和 **15:30** 阶段，Agent 执行多维度评分寻找主线：
1.  **初筛**：调用工具获取近 3/5 日累计涨幅前 5 的板块。
2.  **验真**：过滤涨停家数少、靠权重拉指数的板块，要求主力资金连续 2 日以上净流入。
3.  **共振**：调用 `web_search_news` 确认是否有政策/产业逻辑支撑。无支撑者降级为支线。
4.  **锚定**：将评分最高的 1-2 个板块写入【长期经验记忆】为主线。

### 5.2 介入节点模型
确立主线后，绝不无脑追高。定义三种高性价比/高确定性节点：

*   **节点 A：强分歧日的缩量回踩（左侧）**
    *   逻辑：主线大跌后，次日龙头股缩量企稳不破均线，判定为洗盘。
*   **节点 B：冰点情绪后的共振先行（右侧）**
    *   逻辑：大盘连跌冰点后，主线逆势抗跌，并在大盘企稳瞬间带头放量上攻。
*   **节点 C：轮动补涨的卡位（中继）**
    *   逻辑：龙头连板空间极高时，资金挖掘同板块内滞涨、底部刚放量的中等市值股。

### 5.3 Hermes 动态执行流实战推演
以《主线节点捕捉.skill》注入 Agent 后的实战表现为例：

1.  **周三 15:30**：Agent 发现“半导体”连续流入且有政策催化，锚定为主线。
2.  **周四 10:30**：Agent 盘中发现半导体大幅跳水。Agent 读取 Skill，意识到可能是“强分歧”，未恐慌，规划 11:20 观察龙头封单。
3.  **周四 15:30**：半导体大跌但龙头守住 5 日线。Agent 判定明日可能出现节点 A。
4.  **周五 09:35**：Agent 调用工具发现龙头低开翻红，前 15 分钟量能仅为昨日 50%（极度缩量）。**判定触发高性价比左侧节点！**
5.  **周五 09:36**：Agent 打断常规循环，调用 `schedule_next_wake_up("immediate")`，向用户推送：“⚠️ 触发半导体分歧修复节点！龙头 [XXX] 缩量企稳，建议分时均线介入，止损设昨日最低价。”

## 6. 工程落地与边界控制

### 6.1 调度器边界熔断
*   **午间休市强阻断**：若 Agent 在 11:30-13:00 间错误调用 `schedule_next_wake_up`，底层拦截并推迟至 13:00 后。
*   **尾盘强阻断**：若设定时间 >= 15:00，强制拦截，防止收盘后死循环调用实时盘口接口。

### 6.2 幻觉防范与验证器
在 Agent 输出 `schedule_next_wake_up` 后，增加轻量级代码校验器：
*   检查时间格式是否合法，是否在交易日 09:30-15:00 区间。
*   检查任务间隔，若小于 1 分钟，强制降频为 1 分钟，防止对 API 造成压力。
*   限制每日最大任务数（如 40 个），防止极端行情下 Agent 陷入疯狂刷新死循环。

### 6.3 部署方案
*   使用 Docker 化部署 Hermes Agent 核心服务。
*   接入飞书/企业微信 Bot 作为消息通道，Agent 生成的任务卡片与预警报告自动推送到用户端，用户亦可随时下发盘中指令。
