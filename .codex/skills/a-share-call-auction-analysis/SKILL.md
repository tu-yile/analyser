# A股集合竞价分析

## 目标

在交易日 09:25 集合竞价撮合完成后，判断盘前预期是否获得资金确认，识别一致、分歧和异常信号，为开盘后的观察提供基线。

## 上下文系统

每次运行前，优先读取项目数据系统的精简运行态上下文：

- `data/context/today_state.md`
- `data/context/week_state.md`
- `data/context/recent.md`

必要时再追溯完整上下文：

- `data/context/today.md`
- `data/context/week.md`
- `data/context/recent_index.json`

报告完成后，先将最终报告保存为 Markdown 文件，再运行：

```bash
python3 scripts/market_memory.py add-report --skill call-auction --report <report-file>
```

该命令会把完整报告归档到 `data/history/reports/`，并把完整报告合并到 `data/context/today.md`。

随后基于最新报告、旧 `today_state.md` 和必要的 `week_state.md` 生成新的今日状态文件，再运行：

```bash
python3 scripts/market_memory.py update-today-state --file <new-today-state-file>
```

## 时点纪律

- 若在 09:25 前运行，只输出“竞价观察中”，不得把未撮合委托当作最终开盘价或成交量。
- 若数据在 09:25 后仍未更新，等待最新快照或明确标注延迟。不要用昨日数据冒充今日竞价。
- 非交易日不生成竞价结论，只说明市场休市。

## 工作流

1. 以 `Asia/Shanghai` 核对交易日、当前时间和行情时间戳。
2. 读取当天 `$a-share-overnight-brief` 的结论（若上下文可用），提取待验证线索；没有晨报时自行简要建立外围和消息基线。
3. 优先调用 `$futuapi` 获取指数、ETF、板块和代表性个股的 09:25 快照、昨收、开盘价、成交量额及涨跌幅。需要事件解释时调用 `$futu-news-search`，并用 Web Search 查公告、政策或突发消息。
4. 对指数、市场广度、板块和核心标的分层分析，避免用单只高开股代表整个题材。
5. 明确区分行情事实、新闻催化和分析推断。

## 分析范围

- 指数与风格：上证、深证成指、创业板指、科创 50、沪深 300、中证 1000 或对应高流动性 ETF。
- 市场广度：高开、低开、平开家数，涨停、跌停或接近涨跌停家数；仅在数据源可得且口径清晰时报告。
- 竞价强度：相对昨收缺口、竞价成交量额、相对历史或前一日水平。没有可靠可比口径时只报绝对值。
- 板块联动：同一题材是否有多只成分股同步反馈，核心股与跟随股是否一致。
- 风险信号：指数高开但广度弱、题材高开但核心股低于预期、孤立涨停、重大利空下的异常承接等。

资金流向、主力净流入等供应商指标只能标注为估算口径，不能当作真实买卖主体证据。

## 判断框架

对每条主线给出以下标签之一：

- `超预期`：竞价强度、联动和催化同时得到确认。
- `符合预期`：表现与盘前假设一致，但没有新增强度。
- `低于预期`：关键标的、广度或量能未确认。
- `证据不足`：数据延迟、样本过少或消息尚未核验。

同时写出开盘后 5 至 30 分钟的验证条件和失效条件，不给出确定性涨跌预测。

## 输出格式

### 竞价结论

用 3 至 5 条短句说明指数缺口、市场广度、最强和最弱方向。

### 指数与广度

使用表格列出昨收、开盘、缺口、竞价量额和时间戳；无数据项写明缺失。

### 主线验证

使用表格列出主题、竞价反馈、代表标的、催化来源、判断标签和开盘验证条件。

### 风险与异常

列出最多 5 个背离、孤立异动或数据质量问题。

### 开盘观察清单

给出 3 至 6 个可观察条件，包括量能、回补缺口、核心股承接、板块扩散或指数共振。

最后注明：报告生成时间、行情截止时间、来源链接，以及“仅供市场研究，不构成投资建议”。
