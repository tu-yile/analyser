# A股收盘全日复盘

## 目标

在交易日 15:10 左右完成下午盘面分析和全天总结，回答市场如何收盘、下午相对上午发生了什么变化、当日主线是否强化，以及次日需要验证什么。

## 上下文系统

每次运行前，优先读取项目数据系统的精简运行态上下文：

- `data/context/today_state.md`
- `data/context/week_state.md`
- `data/context/recent.md`

必要时再追溯完整上下文：

- `data/context/today.md`
- `data/context/week.md`
- `data/context/recent_index.json`

报告完成后，先将最终报告保存为 Markdown 文件，再运行：

```bash
python3 scripts/market_memory.py add-report --skill close --report <report-file>
```

该命令会把完整报告归档到 `data/history/reports/`，并把完整报告合并到 `data/context/today.md`。

随后基于最新报告、旧 `today_state.md` 和必要的 `week_state.md` 生成新的今日状态文件，再运行：

```bash
python3 scripts/market_memory.py update-today-state --file <new-today-state-file>
```

收盘报告完成后，如果用户或自动化要求日终/周终归档，使用 `$a-share-context-rollover` 执行独立 rollover 流程。不要在本 skill 内直接调用旧的 `day-rollover` 或 `week-rollover` 命令。

## 时点纪律

- 等待 15:00 收盘价和成交数据稳定后再下结论，标明实际行情时间戳。
- 15:10 前尚未更新的涨跌停、成交额或排行榜数据应标注为暂定。
- 非交易日只复盘最近交易日，不制造“今日盘面”。

## 工作流

1. 以 `Asia/Shanghai` 确认交易日和收盘数据时间戳。
2. 读取当天 `$a-share-overnight-brief`、`$a-share-call-auction-analysis` 和 `$a-share-midday-review`（若上下文可用），建立全天预期演变链。
3. 优先调用 `$futuapi` 获取指数、ETF、板块、个股的收盘快照、日内分时/K线、成交量额和历史对比；需要事件解释时调用 `$futu-news-search` 或 `$futu-stock-digest`。
4. 使用 Web Search 核验当天政策、公告、宏观数据和突发事件。优先原始来源，附链接和发布时间。
5. 对多个数据源的成交额、涨跌家数和涨跌停数量保持单一口径；冲突无法解决时并列说明。

## 必查范围

- 指数：上证、深证成指、创业板指、科创 50、沪深 300、中证 1000 的涨跌、振幅、收盘位置和尾盘表现。
- 成交：沪深两市全天成交额，与前一交易日全天同口径比较；可得时补充近 5 或 20 日位置。
- 广度与情绪：上涨/下跌家数、涨停/跌停、连板晋级、高位股和亏钱效应，注明数据口径。
- 风格：权重/题材、大盘/小盘、成长/价值、红利/科技的相对表现。
- 板块：全天领涨、领跌、午后加强、午后回落和尾盘异动方向。
- 事件：真正发生在盘中的政策、公告或外部变量，区分驱动证据与时间上的巧合。

资金流向和主力行为只能作为供应商估算信号，不能据此断言具体机构正在买入或卖出。

## 分析框架

- 将下午与上午比较：指数方向、量能斜率、市场广度、主线扩散和核心标的承接发生了什么变化。
- 将收盘与盘前、竞价和午间预期比较，形成 `预期 → 竞价 → 上午 → 下午 → 收盘` 的证据链。
- 判断主线状态为 `强化 / 分化 / 退潮 / 轮动 / 无清晰主线`，并列出证据。
- 判断市场结构为 `偏强 / 中性 / 偏弱`，但避免以单日结果推断长期趋势。
- 提炼次日观察点，包括延续条件、修复条件和风险失效条件，不给出确定性预测或买卖指令。

## 输出格式

### 收盘结论

用 3 至 5 条短句总结全天最重要的结构和下午变化。

### 全日数据

使用表格列出指数、成交额、同口径变化、市场广度、涨跌停和时间戳。

### 下午变化

明确比较 11:30 与 15:00 的指数、量能、板块和情绪变化。

### 主线与风格

列出主线状态、代表标的、催化、扩散程度、持续性证据和主要风险。

### 全天预期复盘

按时间线总结哪些盘前假设得到验证、何时被证伪，以及新信息如何改变市场定价。

### 次日观察清单

给出 3 至 6 个可验证条件，覆盖指数、量能、主线、核心标的和外部变量。

最后注明：报告生成时间、行情截止时间、数据缺口、来源链接，以及“仅供市场研究，不构成投资建议”。
