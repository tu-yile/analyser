# A股午间盘面复盘

## 目标

在交易日 12:30 左右复盘 09:30-11:30 的真实盘面，解释资金实际选择了什么，并给出下午可验证的情景，不使用午间消息对上午走势进行事后硬套。

## 上下文系统

每次运行前，优先读取项目数据系统的精简运行态上下文：

- `data/context/today_state.md`
- `data/context/week_state.md`
- `data/context/recent.md`

必要时再追溯完整上下文：

- `data/context/today.md`
- `data/context/week.md`
- `data/context/recent_index.json`

报告完成后，先将最终报告保存为 Markdown 文件，再运行：

```bash
python3 scripts/market_memory.py add-report --skill midday --report <report-file>
```

该命令会把完整报告归档到 `data/history/reports/`，并把完整报告合并到 `data/context/today.md`。

随后基于最新报告、旧 `today_state.md` 和必要的 `week_state.md` 生成新的今日状态文件，再运行：

```bash
python3 scripts/market_memory.py update-today-state --file <new-today-state-file>
```

## 工作流

1. 以 `Asia/Shanghai` 确认交易日，并标注行情截止时间应为 11:30 左右。非交易日不生成盘中结论。
2. 读取当天 `$a-share-overnight-brief` 和 `$a-share-call-auction-analysis` 的结论（若上下文可用），列出已验证、被证伪和仍待验证的假设。
3. 优先调用 `$futuapi` 获取指数、ETF、板块、个股快照和分时/K线；需要公司或主题催化时调用 `$futu-news-search` 或 `$futu-stock-digest`。
4. 使用 Web Search 核验上午发布的公告、政策、宏观数据和突发事件，优先原始来源并附发布时间与链接。
5. 对无法获得的市场广度、涨跌停、同时间量能对比或资金流数据明确标注缺失，不用不同口径数据拼接。

## 必查范围

- 指数：上证、深证成指、创业板指、科创 50、沪深 300、中证 1000 的涨跌、振幅和分时结构。
- 量能：沪深两市上午成交额；优先与前一交易日 11:30 同口径比较，不得拿半日成交额直接对比前一日全天。
- 广度与情绪：上涨/下跌家数、涨停/跌停、连板或高位股反馈，仅在口径可靠时使用。
- 风格：大盘/小盘、价值/成长、红利/科技、权重/题材之间的相对强弱。
- 板块：领涨、领跌、冲高回落和低位扩散方向，结合代表标的和催化判断持续性。
- 关键资产：人民币、A50 或其他与盘中变化直接相关的外部变量。

北向交易、主力资金等数据若没有实时公开或稳定口径，应省略或明确标注为供应商估算。

## 分析框架

- `走势结构`：单边、震荡、V形、冲高回落或探底回升。
- `量价关系`：放量上涨、缩量上涨、放量下跌或缩量调整。
- `主线质量`：指数共振、板块扩散、核心标的承接和消息持续性。
- `预期校验`：盘前与竞价判断哪些成立，哪些被证伪，原因是什么。
- `下午情景`：给出基准、偏强、偏弱三种可观察情景及触发条件，不分配主观概率，不给买卖指令。

## 输出格式

### 午间结论

用 3 至 5 条短句概括上午最重要的结构变化。

### 市场概览

使用表格列出指数表现、上午成交额、同口径变化、市场广度和时间戳。

### 板块与风格

列出领涨、领跌、冲高回落和逆势方向，说明代表标的、催化与持续性证据。

### 盘前假设复核

使用 `已验证 / 被证伪 / 待验证` 分类整理。

### 下午观察

列出基准、偏强、偏弱情景的触发条件，以及 3 至 6 个关键观察位或信号。

最后注明：报告生成时间、行情截止时间、数据缺口、来源链接，以及“仅供市场研究，不构成投资建议”。
