# A股日内复盘 Skill 改进清单

目标文件：`SKILL.md`

使用说明：

- 按阶段依次修改，优先解决会产生错误行为的问题。
- 每完成一项，同时检查其“验收标准”。
- 除非一项改动依赖另一项，否则一次只修改一个主题，便于验证。

## 阶段一：统一操作建议要求（已完成）

### 1. 统一操作建议边界

- [x] 删除“不给买卖指令”与“给出操作建议”之间的冲突，明确报告需要给出操作建议。
- [x] 操作建议可以覆盖板块、龙头标的、参与条件、仓位节奏和回避方向。
- [x] 每条操作建议必须同时说明盘面依据、触发条件、失效条件和主要风险。
- [x] 区分当日盘中、收盘后和历史回溯场景，避免使用事后行情生成站在历史时点不可能得到的建议。
- [x] 禁止承诺收益或使用“必涨”“确定性机会”等无法验证的表达。
- [x] 在报告末尾保留“仅供市场研究，不构成投资建议”。

验收标准：全文明确要求输出操作建议；每项建议都有依据、触发条件、失效条件和风险说明，不包含收益承诺。

## 阶段二：完善日期和时间逻辑

### 2. 增加交易日校验

- [ ] 使用交易日历确认目标日期是否为A股交易日。
- [ ] 用户只给出月日时，明确年份推断规则。
- [ ] 用户未指定日期且当天休市时，明确是停止还是回退至最近交易日。
- [ ] 区分正常交易日、半日市和临时休市情况。

验收标准：周末、节假日和历史日期均能得到确定且可解释的处理结果。

### 3. 重写默认分析区间

- [ ] 修复“上次分析分析时间（如果没有则）”病句和缺失逻辑。
- [ ] 无上次分析时默认从 `09:30` 开始。
- [ ] 有上次分析时，明确采用“开盘至当前”还是“上次分析后增量”。
- [ ] 建议主报告保持“开盘至当前”，另列上次分析后的变化。
- [ ] 盘前运行时不生成空日内复盘。
- [ ] 午休期间截止 `11:30`，收盘后截止 `15:00`。

验收标准：任意运行时间都能唯一确定开始和截止时点。

### 4. 区分三种报告场景

- [ ] 当日盘中：输出当日下午或剩余交易时段情景。
- [ ] 当日收盘后：输出下一交易日观察框架。
- [ ] 历史回溯：站在历史截止时点给出后续情景，禁止混入当前日期的“明天”。

验收标准：报告中的“后续”始终相对于目标日期和行情截止时点。

### 5. 消除“上午”硬编码

- [ ] 将“上午成交额”改为“截至分析时点的累计成交额”。
- [ ] 将“核验上午事件”改为“核验分析区间内及此前可能影响盘面的事件”。
- [ ] 将市场概览中的固定午间字段改为动态截止时点。
- [ ] 与前一交易日相同截止时点比较，不固定使用 `11:30`。

验收标准：Skill可正确处理午间、下午盘中和收盘复盘。

## 阶段三：统一数据口径

### 6. 定义指数指标算法

- [ ] 涨跌幅统一相对前收盘价计算。
- [ ] 振幅统一定义为 `(区间最高价 - 区间最低价) / 前收盘价`。
- [ ] 明确分钟线起止边界以及截止分钟是否计入。
- [ ] 明确指数行情使用不复权口径。
- [ ] 记录区间高低点及其发生时间。

验收标准：相同数据输入能够稳定得到相同结果。

### 7. 定义成交额口径

- [ ] 确认上证指数和深证成指分钟成交额字段能否代表沪深市场成交额。
- [ ] 明确累计成交额的计算方法和单位换算。
- [ ] 同比对象必须是前一交易日相同截止时点。
- [ ] 无同口径历史数据时明确标注缺失，不使用全天数据替代。

验收标准：报告不会出现半日与全天、不同截止时点之间的错误比较。

### 8. 定义市场广度股票池

- [ ] 明确统计沪深A股还是全A市场。
- [ ] 明确是否包含北交所、B股、ST、退市整理股、新股和停牌股。
- [ ] 上涨、下跌、平盘家数必须来自同一供应商和同一股票池。
- [ ] 无法确认股票池时只保留定性描述并注明来源口径。

验收标准：市场广度的分母和排除项清晰可追溯。

### 9. 定义涨跌停和情绪口径

- [ ] 严格涨跌停数量不得用 `>=9.5%` 或 `<=-9.5%` 近似值冒充。
- [ ] 明确是否包含ST、退市股和新股。
- [ ] 连板数量、炸板率、封板率必须使用同一统计源。
- [ ] 不同来源存在差异时分别列示，不合并为单一精确数字。

验收标准：所有情绪数据都带有可解释的统计口径。

### 10. 规范外部变量使用

- [ ] 人民币、A50、港股、商品和海外期货改为条件必查项。
- [ ] 只有与A股出现时间同步或逻辑直接相关时才进入正文。
- [ ] 无权限或无同频数据时明确标注缺失。
- [ ] 禁止使用日线或延迟报价解释分钟级盘中拐点。

验收标准：外部变量用于验证盘面，不作为装饰性数据堆砌。

## 阶段四：强化分析框架

### 11. 区分事实、推断和情景

- [ ] “事实”只写价格、成交、广度、公告和事件发布时间。
- [ ] “推断”说明根据哪些盘面证据得出。
- [ ] “情景”必须包含触发条件和失效条件。
- [ ] 避免把“市场资金选择”写成已知事实。

验收标准：读者能够区分数据、分析者判断和未来条件。

### 12. 建立主线质量标准

- [ ] 检查指数是否共振。
- [ ] 检查板块上涨广度和涨停扩散。
- [ ] 检查核心标的承接及是否冲高回落。
- [ ] 检查成交额是否支持行情延续。
- [ ] 检查强势持续时间，而非只看某一时点涨幅。
- [ ] 检查催化的可信度和后续验证节点。

验收标准：主线判断不再只依赖涨幅排名或单条消息。

### 13. 增加指数与赚钱效应分离判断

- [ ] 对比宽基指数与市场广度。
- [ ] 区分权重贡献和多数个股表现。
- [ ] 条件允许时加入等权表现或涨跌幅中位数。
- [ ] 明确识别“指数强、个股弱”或“题材强、权重弱”。

验收标准：不会把少数权重上涨误判为全面回暖。

### 14. 增加分时关键节点分析

- [ ] 识别开盘缺口和开盘后首次高低点。
- [ ] 记录主要指数及主线板块的同步拐点。
- [ ] 午间报告至少检查 `09:30`、`10:00`、`11:30` 附近结构。
- [ ] 收盘报告增加 `13:30`、`14:30` 和尾盘结构。

验收标准：走势结构判断有明确时间证据，不只依赖午收或收盘涨跌幅。

### 15. 完善预期复核

- [ ] 只复核项目中真实留存的盘前、竞价或上次日内判断。
- [ ] 没有历史基线时写“无可复核基线”。
- [ ] 禁止根据实际走势反向补造盘前假设。
- [ ] 每项判断注明验证证据或证伪原因。

验收标准：预期复核可追溯到已有报告或上下文原文。

### 16. 规范催化分析

- [ ] 优先使用交易所、公司公告、政府和监管机构原始来源。
- [ ] 区分已确认事实、媒体报道、券商观点和市场传闻。
- [ ] 记录事件发布时间，确认其早于相应盘面变化。
- [ ] 将“消息持续性”改为“催化可信度、定价程度和后续验证节点”。
- [ ] 不用消息对走势进行唯一归因。

验收标准：催化能够解释市场关注点，但不会替代盘面证据。

## 阶段五：优化输出和归档

### 17. 精简并固定报告结构

- [ ] 结论保持3至5条，每条只表达一个核心判断。
- [ ] 市场概览动态使用实际截止时点。
- [ ] 板块部分固定包含领涨、领跌、冲高回落、逆势和近期主线复核。
- [ ] 预期复核允许输出“无基线”。
- [ ] 后续观察包含基准、偏强、偏弱情景及关键触发条件。
- [ ] 末尾固定列出生成时间、截止时间、缺口、来源和免责声明。

验收标准：报告结构稳定，但不会因为缺失数据而填充虚假内容。

### 18. 改写含糊或冗长表述

- [ ] 拆分板块必查范围中的长句。
- [ ] 为“近期”“有异动”“辨识度”“必要时”等词给出可执行定义。
- [ ] 将“特殊的”改为“特殊情况”。
- [ ] 统一使用“科创50”“沪深300”等名称的空格格式。
- [ ] 修复重复词、缺少标点和不完整句子。

验收标准：每一条指令都能直接转化为具体动作或判断。

### 19. 调整报告文件命名

- [ ] 明确文件名中的时间是行情截止时间还是生成时间。
- [ ] 避免同一截止时点重复运行时静默覆盖旧报告。
- [ ] 可采用 `day-review-${cutoff-HHmm}-${generated-HHmmss}.md`。
- [ ] 若保持原命名，明确覆盖、备份或版本化规则。

验收标准：重复运行不会造成无法察觉的数据丢失。

### 20. 增加保存失败和部分完成处理

- [ ] 报告生成成功后再更新上下文。
- [ ] 报告保存失败时不得更新 `today.md`。
- [ ] 上下文更新失败时保留报告并明确提示。
- [ ] 不覆盖用户已有的无关修改。

验收标准：报告与上下文不会出现一半成功、一半失败而未说明的状态。

## 最终回归检查

- [ ] 当日10:30运行：生成开盘至10:30复盘和剩余时段情景。
- [ ] 当日12:00运行：截止11:30，并与前一交易日11:30比较。
- [ ] 当日14:00运行：与前一交易日14:00同口径比较。
- [ ] 当日收盘后运行：输出下一交易日观察。
- [ ] 周末运行且未指定日期：行为符合非交易日规则。
- [ ] 指定历史日期运行：能够完整生成报告并按现有规则更新 `today.md`，用于历史交易日测试。
- [ ] 无市场广度数据：明确缺失，不拼接其他口径。
- [ ] 无盘前报告：输出“无可复核基线”。
- [ ] 外部行情权限不足：报告保留缺口，不用替代数据硬凑。
- [ ] 重复运行相同截止时点：旧报告不会被静默覆盖。

## 建议实施顺序

1. 完成阶段一，统一操作建议的内容和风险约束。
2. 完成阶段二，建立明确的日期与时间状态机。
3. 完成阶段三，固定数据口径。
4. 完成阶段四，提升分析质量和可验证性。
5. 完成阶段五，精简输出并保证归档可靠。
6. 执行全部回归检查，再将清单标记为完成。
