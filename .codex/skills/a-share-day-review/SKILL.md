# A股日内复盘

## 目标

先陈述可核验事实，再判断市场状态、主线质量和操作条件。输出必须帮助用户回答：

- 当前属于进攻、结构性参与、防守、风险收缩还是无交易条件。
- 哪些方向可参与、仅观察或应回避。
- 什么信号出现后提高或降低参与度。
- 用户现有持仓与市场主线是否一致，哪些应持有、降低暴露或等待确认。

不得把推断写成资金流事实，不得用消息事后硬套走势，不得为了给建议而强行寻找机会。

## 上下文

运行前读取：

- `data/context/today.md`
- `data/context/week.md`
- `data/context/recent.md`
- `data/myStock/data.json`

盘中只强调相较上次分析新增、强化、降级或失效的变化；不重复无变化的旧结论。

### 持仓文件

默认结构：

```json
{
  "hold": [
    {"name": "股票名称或代码", "position": 10.0}
  ]
}
```

- 将 `position` 解释为该标的占组合的百分比权重。
- 校验名称、权重和总和；权重缺失、非数字、负数或总和超过100时明确报错，不自行修正。
- 权重总和低于100时，只报告“已列持仓合计”和“未列明部分”，不得擅自认定剩余部分为现金。
- 文件缺失、为空或没有有效持仓时，省略持仓章节并注明未取得持仓数据。
- 文件未提供成本价时，不计算持仓盈亏，不使用“回本价”或基于成本价的止损条件。

## 工作流

1. **确认交易日与时间范围**
   - 优先用 `$futuapi` 交易日历校验目标日期，记录全天、半日市或休市状态。
   - 用户未指定日期且当天非交易日时，回退到最近交易日并说明原因；用户明确指定非交易日时停止复盘。
   - 未指定时间范围时，从 `today.md` 的上次分析时间开始；无记录时从 `09:30` 开始，截止到最新可得行情。

2. **全市场宽筛**
   - 获取六大宽基指数、同口径成交额、市场广度和风格代理。
   - 构造行业涨跌幅、概念涨跌幅、板块成交额、个股涨跌幅和个股成交额排行；必要时取得涨跌停近似样本。
   - 用多类排行中的重复主题和板块—个股交叉验证形成领涨、领跌、高成交、冲高回落和逆势候选。
   - 不得因某方向已存在于上下文而跳过宽筛或直接认定其仍是主线。

3. **读取并映射持仓**
   - 将持仓名称规范化为富途标准代码，获取每只持仓的快照、所属板块、相对相关宽基和相关板块的强弱。
   - 将持仓归类到本次宽筛形成的主线、候选、分歧或弱势方向；不得脱离全市场结构孤立评价个股。
   - 汇总同主题或高相关持仓的权重，识别组合对单一产业链、风格和指数的集中暴露。
   - 每只持仓至少检查截止时点快照。满足任一条件时检查5分钟分时：权重不低于5%、进入涨跌幅或成交额前列、所属方向状态变化、个股相对板块明显背离、操作建议可能发生变化。

4. **先判定市场状态**
   - 综合指数结构、同口径量能、广度与跌停扩散、主线数量和质量，从以下状态中选择一个：
     - `积极进攻`：指数、量能、广度和主线共同确认。
     - `结构性参与`：存在有效主线，但量能、广度或权重存在明显反证。
     - `防守轮动`：指数或广度偏弱，强势集中于防守方向。
     - `风险收缩`：关键指数破位、广度恶化或跌停扩散，主线同步失效。
     - `无明确交易条件`：证据冲突或没有满足条件的方向。
   - 证据冲突时选择更保守的状态。

5. **轻量验证与分时下钻**
   - 六大指数强制扫描完整分析区间的5分钟K线。
   - 宽筛候选先使用板块或2只代表标的的近期5分钟结构做轻量验证；确认需要判断持续性、拐点或同步性后再扫描完整区间。
   - 当日辨识度板块、影响核心结论的领跌方向和状态可能升级或失效的近期主线需要完整下钻。
   - 普通候选、无异动的历史主线和长尾方向使用快照即可。
   - 每个方向默认选择2至3只代表标的：龙头或涨停核心、高成交核心、必要时一只反证标的。
   - 板块分时不可得时使用ETF或代表标的组合替代并说明口径。
   - 持仓标的已触发分时检查时可兼作该方向的代表或反证样本，避免重复拉取。

6. **核验事件**
   - 使用 Web Search 核验分析截止时点前的政策、公告、宏观数据和突发事件，优先原始来源并附时间与链接。
   - 催化只验证盘面逻辑，不得覆盖价格、广度或核心标的已经给出的失效信号。

7. **形成决策与更新上下文**
   - 先生成交易决策卡，再补充市场事实、分时证据和情景。
   - 持仓建议必须服从市场风险状态和方向状态，不能因用户已持有而降低失效标准。
   - 最终报告保存为 `data/history/reports/${YYYY-MM-DD}/day-review-${HH:mm}.md`。
   - 按 `data/context/today.md` 的规则更新运行态上下文。

## 数据与分析要求

### 必查数据

- 指数：上证指数、深证成指、创业板指、科创50、沪深300、中证1000。
- 量能：沪深两市累计成交额，与前一交易日相同市场、相同截止时间、相同数据源比较。
- 广度：上涨/下跌家数；涨跌停、连板和高位股反馈仅在口径可靠时使用。
- 风格：权重/题材、大盘/小盘、价值/成长、红利/科技。
- 板块与个股：行业和概念涨跌幅、板块成交额、个股涨跌幅和成交额排行。
- 持仓：持仓权重、所属方向、相对板块与宽基表现、分时结构和主题集中度。
- 外部资产：人民币、A50等只有在与A股盘中结构存在可观察同步或背离时检查。

无法取得的数据必须标注缺口，不拼接不同口径。北向交易、主力资金等没有稳定口径时省略或注明供应商估算。

### 方向状态

每个重要方向只标记一个当前状态：

`观察候选 → 当日辨识度方向 → 已确认主线 → 分歧/降级 → 失效`

- `当日辨识度方向`：在可得口径中至少满足两项：
  - 上涨家数多于下跌家数。
  - 同截止时点成交额高于前一交易日。
  - 至少两只代表标的同向且显著强于或弱于相关宽基。
  - 至少三只严格涨停或跌停。
  - 板块与相关宽基连续两个有结构意义的分时阶段同向。
- 不满足以上条件，但有一只核心涨停或两只代表标的明显强于宽基时，记录为候选或局部强分支，并列出反证。
- 近期定义为目标日及此前5个交易日。只在状态发生变化时重点展开。

### 信号优先级

决策优先级固定为：

`市场风险状态 > 组合集中风险 > 板块扩散与成交 > 持仓/核心标的反馈 > 催化 > 单股异动`

硬约束：

- 广度显著恶化且跌停扩散时，不得给出“积极进攻”。
- 缩量且方向只剩单只核心时，最多定义为观察候选。
- 板块上涨但高成交核心不确认时，不得升级为已确认主线。
- 多只持仓属于同一高相关方向时，必须按合并暴露评估风险，不得把每只股票分别视为独立分散。
- 持仓个股走强但所属方向失效时，不得仅凭单股强势给出增加暴露建议。
- 单条消息或单股异动不得推翻市场风险状态和板块证据。
- 没有满足条件的机会时，明确写“当前无交易条件”。

### 可观察条件

尽量把“承接、破位、快速回落、封单稳定”等表述转换为可验证信号，例如：

- 5分钟收盘是否跌破分时均价或上一有效回踩低点。
- 突破后是否连续两个5分钟阶段站稳，或快速跌回突破位下方。
- 板块涨幅是否回吐超过主要上行段的一半。
- 两只高成交核心是否同步跌破昨收或关键分时低点。
- 涨停开板后两个5分钟周期内能否重新封板。

只有在数据不足以量化时才使用定性描述，并说明判断依据。

## 操作框架

### 暴露等级

不机械给出投资比例，使用统一等级：

- `回避`：市场处于风险收缩，或方向已失效。
- `观察仓`：只有局部证据，等待确认。
- `结构性参与`：主线成立，但广度、量能或权重存在反证。
- `积极参与`：市场与主线多维度共同确认。

若市场信号与板块信号冲突，以较低等级为准。

### 操作建议

最多给出3条最重要的建议。每条必须包含：

- `动作`：参与、等待回踩、持有观察、降低暴露或回避。
- `参与条件`：可直接观察和判断的触发信号。
- `失效条件`：出现后应放弃或调整的信号。
- `主要风险`：最可能导致判断失败的因素。

用户提供持仓时，增加持仓映射：顺应主线、主线分歧、非主线未破位、与市场风险共振走弱；未提供持仓时不虚构持仓建议。

### 持仓动作

每只持仓使用以下动作之一：

- `持有`：所属方向有效，个股不弱于方向，且没有触发组合集中风险。
- `持有观察`：方向有效但个股承接不足，或个股有效但市场存在明显反证。
- `等待确认后增加`：市场、方向和个股触发条件尚未同时满足；不得只因浮盈或浮亏建议增加。
- `降低暴露`：方向降级、个股明显弱于板块、关键结构失效，或组合在同一高相关方向过度集中。
- `退出条件已触发`：市场、方向和个股失效信号已经成立。只描述研究框架，不代替用户下单。

动作必须包含可观察的触发或失效条件。没有成本价时，条件只能基于价格结构、相对强弱、成交和方向状态。

## 输出格式

### 交易决策卡

置于报告最前，使用表格：

| 项目 | 结论 |
| --- | --- |
| 市场状态 | 五类状态之一 |
| 整体暴露 | 回避 / 观察仓 / 结构性参与 / 积极参与 |
| 可参与方向 | 最多3个；无则写无 |
| 仅观察方向 | 等待确认的候选 |
| 回避方向 | 当前明显弱势或失效方向 |
| 持仓风险摘要 | 已列持仓合计、主要主题暴露和最高优先级风险 |
| 提高参与条件 | 市场级确认信号 |
| 降低参与条件 | 市场级风险信号 |
| 是否适合追涨 | 是 / 否，并给一句依据 |

### 市场概览

只列事实：六大指数、成交额同口径变化、市场广度、行情截止时间。分析层扫描六个指数，输出层优先聚合为：

- 权重：上证、沪深300。
- 成长：创业板、科创50。
- 中小盘：深成指、中证1000。

只有明显背离时单列指数。

### 本次新增分时结构

按实际结构划分阶段，只写相较上次分析新增的走势和已确认拐点；首次或收盘复盘再概括完整结构。比较指数组、已下钻主线和核心标的的同步、领先、滞后或背离，不逐根复述。

### 方向状态表

| 方向 | 当前状态 | 核心证据与反证 | 参与条件 | 失效条件 | 动作 |
| --- | --- | --- | --- | --- | --- |

只展开对决策有影响的方向，通常不超过5个。未触发分时下钻时注明使用截止时点快照，不得推断全天持续性。

### 持仓分析

先给组合摘要：

- 已列持仓总权重和未列明权重。
- 主要主题/产业链的合计暴露。
- 与当前市场主线、风格和风险状态的匹配程度。
- 最大单一风险和需要优先处理的信号。

再使用表格逐只分析：

| 持仓 | 权重 | 所属方向 | 相对强弱与结构 | 动作 | 增加/继续持有条件 | 降低暴露/失效条件 |
| --- | ---: | --- | --- | --- | --- | --- |

- 按权重和风险优先级排序，不按股票代码排序。
- 方向高度相关的持仓应明确合并暴露，不得重复给出互相独立的增加建议。
- 若所有持仓均属于同一风格或产业链，必须单列集中度警告。
- 持仓操作建议不计入市场级“最多3条操作建议”的数量限制，但每只持仓只保留一个主要动作。

### 预期校验

只写本次新增的 `已验证 / 被证伪 / 待验证` 项；不重复未变化内容。

### 市场情景

保留基准、偏强、偏弱三种市场级情景，各写触发和失效条件。不要在此重复方向状态表中的全部个股证据。

### 操作建议

最多3条市场级建议，按“动作—参与条件—失效条件—风险”输出。持仓动作放在持仓分析表中，不在此重复。若决策卡已充分表达，不重复盘面事实。

### 元信息

注明报告生成时间、行情截止时间、数据缺口、来源链接及“仅供市场研究，不构成投资建议”。

盘中报告以100至150行为目标；收盘报告可适当增加完整结构，但避免重复盘中已有事实。
