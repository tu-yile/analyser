# A股日内复盘

## 目标

在交易日日内对过去的盘面进行回顾分析，先说明可核验的盘面事实，再根据价量、广度和扩散结构推断市场的主要定价方向，并给出后续可验证的情景。不要把推断写成资金流事实，也不要使用消息对盘面走势进行事后硬套。

## 上下文系统

每次运行前，优先读取项目数据系统的运行态上下文：

- `data/context/today.md`
- `data/context/week.md`
- `data/context/recent.md`

## 工作流

1. 确认目标日期是否为A股交易日，再确定分析时间范围（最早为 `09:30`，最晚为对应交易时段的收盘时间）。
  - 优先调用 `$futuapi` 交易日历校验目标日期，并记录正常交易日、半日市或临时休市状态。
  - 如果用户指定了完整日期和时间则以用户指定的为准；如果只指定月日，则取 `Asia/Shanghai` 当前日期之前或当天最近的同月同日作为目标日期。
  - 如果用户明确指定的日期不是交易日，不得自动替换为其他日期；停止生成盘面复盘，并说明交易日历结果。
  - 如果用户没有指定日期且当前日期不是交易日，则回退到最近一个A股交易日，并在报告中注明日期回退及原因。
  - 遇到半日市时，以交易日历返回的实际交易时段确定截止时间和同口径比较区间；遇到临时休市或行情缺失时，明确说明，不把无交易数据解释为缩量。
  - 如果用户没有指定时间范围，则读取 `data/context/today.md` 获取目标日期的上次分析时间；没有上次分析记录时从 `09:30` 开始，否则从上次分析时间开始，截止到当前可获得的最新行情时间。
2. 优先调用 `$futuapi` 获取指数、ETF、板块、个股快照和分时/K线，具体如何分析参考后面的《必查范围》和《分析框架》；需要公司或主题催化时调用 `$futu-news-search` 或 `$futu-stock-digest`。
3. 使用 Web Search 核验分析区间内及此前可能影响盘面的公告、政策、宏观数据和突发事件，优先原始来源并附发布时间与链接。不得使用行情截止时点之后发布的信息解释此前走势。
4. 对无法获得的市场广度、涨跌停、相同截止时点量能对比或资金流数据明确标注缺失，不拼接不同口径的数据。

## 必查范围

- 指数：上证指数、深证成指、创业板指、科创50、沪深300、中证1000的涨跌、振幅和完整分时结构；扫描整个分析区间的分钟K线，记录区间高低点、有效拐点及其发生时间。
- 量能：统计沪深两市截至分析截止时点的累计成交额，并与前一交易日相同截止时点、相同市场范围和相同数据源比较。不得使用不同截止时点或全天数据替代。
- 广度与情绪：上涨/下跌家数、涨停/跌停、连板或高位股反馈，仅在口径可靠时使用。
- 风格：大盘/小盘、价值/成长、红利/科技、权重/题材之间的相对强弱。
- 板块：
  - 检查领涨、领跌、冲高回落、逆势走强和低位扩散方向。
  - 将至少满足以下两项的方向定义为“当日辨识度板块”：上涨家数多于下跌家数；截至同一时点的成交额高于前一交易日；至少两只代表标的同向运行且相对相关宽基指数更强或更弱；出现至少三只严格涨停或跌停；板块与相关宽基指数在连续两个分时阶段同向运行。仅使用口径可靠的可得指标，缺失项不得视为满足。
  - 将“近期”定义为目标交易日及此前5个交易日。结合 `today.md`、`week.md` 和 `recent.md`，复核在该窗口内曾满足上述至少两项条件，或已被运行态上下文记录为主线的板块。
  - 对当日辨识度板块和近期主线检查完整5分钟分时；若板块分时不可得，则使用板块指数、ETF或多只代表标的交叉验证并说明替代口径。
  - 根据代表标的承接、板块扩散、成交持续性和催化验证节点判断持续性。
- 关键资产：人民币、A50 或其他与盘中变化直接相关的外部变量。

北向交易、主力资金等数据若没有实时公开或稳定口径，应省略或明确标注为供应商估算。

## 分析框架

- `走势结构`：单边、震荡、V形、冲高回落或探底回升。
- `量价关系`：放量上涨、缩量上涨、放量下跌或缩量调整。
- `分时结构`：扫描整个分析区间内的5分钟K线。根据价格方向、量能变化、结构突破或跌破以及指数与板块共振，识别上涨、回落、横盘和反转阶段。只有在后续走势形成明确方向延续、结构突破或结构失效时，才将局部高低点认定为有效拐点并说明确认依据；截至行情截止时点仍未确认的拐点标记为“待确认”。比较主要指数、主线板块和核心标的的完整轨迹，识别同步、领先、滞后和背离。完整扫描不等于逐根复述，应合并没有结构意义的短时噪声；只使用不晚于行情截止时点的数据。
- `证据分层`：将内容分为事实、推断和情景。事实仅包括可直接核验的价格、成交、市场广度、公告内容和事件发布时间；推断必须指出所依据的盘面事实；情景必须同时包含触发条件和失效条件。
- `主线质量`：依次检查指数共振、板块上涨广度与涨停扩散、核心标的承接与冲高回落、成交额支持、强势持续时间、催化可信度及后续验证节点。对可获得的维度同时列出支持证据和反证；数据缺失时明确标注，不得仅凭涨幅排名或单条消息认定主线。
- `预期校验`：盘前与竞价判断哪些成立，哪些被证伪，原因是什么。
- `后续情景`：给出基准、偏强、偏弱三种可观察情景，并分别说明触发条件和失效条件，不分配主观概率。
- `操作策略`：基于盘面证据和后续情景，给出板块、龙头标的、参与条件、仓位节奏和回避方向的建议。每条建议必须说明盘面依据、触发条件、失效条件和主要风险；历史回溯只能使用行情截止时点前已经存在的信息。禁止承诺收益或使用“必涨”“确定性机会”等无法验证的表达。

## 输出格式

### 结论

用 3 至 5 条短句概括分析的时间范围内盘面上最重要的结构变化。

### 市场概览

只列可直接核验的事实。使用表格列出指数表现、截至分析截止时点的累计成交额、与前一交易日相同截止时点的变化、市场广度和行情截止时间。

### 分时结构

按行情实际演变划分分时阶段，不使用预设时间点切割走势。逐段说明起止时间、价格方向、量能变化、区间高低点和有效拐点，并比较主要指数、主线板块与核心标的是同步、领先、滞后还是背离。每个关键点都要给出时间、价量证据和有效性依据；不得只依据午收、收盘涨跌幅或事后结果判断单边、V形、冲高回落等结构。

### 板块与风格

列出领涨、领跌、冲高回落和逆势方向。每个方向分开说明：

- `盘面事实`：代表标的、板块广度、成交、持续时间和分时反馈。
- `结构判断`：基于上述事实判断主线质量、持续性以及支持证据和反证。

催化只作为验证维度，不得替代盘面证据。

### 盘前假设复核

使用 `已验证 / 被证伪 / 待验证` 分类整理。

### 后续观察与操作策略

列出基准、偏强、偏弱情景的触发条件和失效条件，以及 3 至 6 个关键观察位或信号。特殊情况：如果分析时间是在当日收盘后，则给出明天整个交易日的观察。

在情景分析后给出可执行的操作建议，可以覆盖板块、龙头标的、参与条件、仓位节奏和回避方向。每条建议都要包含：

- `盘面依据`：支持该建议的指数、量能、板块扩散或核心标的证据。
- `触发条件`：什么情况出现后建议才成立。
- `失效条件`：什么情况出现后应放弃或调整建议。
- `主要风险`：可能导致判断失败的盘面或事件因素。

当分析历史交易日时，操作建议必须站在行情截止时点生成，不得引用该时点之后已经发生的走势、公告或结果。

最后注明：报告生成时间、行情截止时间、数据缺口、来源链接，以及“仅供市场研究，不构成投资建议”。

## 保存并归档

最终报告生成一份 Markdown 文件，并保存为 `data/history/reports/${YYYY-MM-DD}/day-review-${HH:mm}.md`

然后基于这份分析报告并按照`data/context/today.md`的规则对 `data/context/today.md`进行更新，为当天后续的盘面分析提供基础

如果分析范围的终止在当日的收盘时间后（15:00及以后），则结合`data/context/today.md`内容以及本次分析报告，对`data/context/week.md`进行更新，之后再清空`data/context/today.md`的内容部分
