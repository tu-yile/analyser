# A股隔夜外围与重大事件晨报

## 目标

在北京时间 `08:00` 左右复盘前一A股交易日收盘后至晨报截止时点的全球市场和重大事件，先说明可核验的跨资产事实，再根据价格、波动、相关性和事件反应推断隔夜主导变量，最后形成当日A股开盘、风格和行业的可验证情景。不要把推断写成资金流事实，不要把外围涨跌或单一新闻机械等同于A股方向，也不要使用截止时点之后的信息解释此前市场反应。

## 上下文系统

每次运行前，优先读取项目数据系统的运行态上下文：

- `data/context/today.md`
- `data/context/week.md`
- `data/context/recent.md`

## 工作流

1. 确认目标日期是否为A股交易日，再确定分析窗口和报告用途。
  - 优先调用 `$futuapi` 交易日历校验目标日期，并记录正常交易日、半日市或临时休市状态。
  - 如果用户指定完整日期则以用户指定日期为目标日期；如果只指定月日，则取 `Asia/Shanghai` 当前日期之前或当天最近的同月同日。
  - 如果用户明确指定的日期不是交易日，不得自动替换；仍可复盘该日期之前的最近完整海外交易时段，但必须明确该报告不对应A股当日开盘，并给出下一A股交易日的待验证风险。
  - 如果用户未指定日期且当前日期不是A股交易日，则复盘最近完整海外交易时段，并将情景映射到下一A股交易日，注明日期回退或顺延及原因。
  - 默认分析窗口从前一A股交易日收盘时间开始，至当前可获得的最新行情与新闻时间；用户指定时间范围时以其为准。
  - 遇到海外休市、半日市、夏令时切换、临时停牌或行情缺失时，明确实际交易时段和数据状态，不把无交易数据解释为市场平静或缩量。
2. 分别标注行情截止时间和新闻检索截止时间。美国市场未收盘、期货仍交易、亚太市场尚未开盘或关键数据尚未公布时，明确写明“盘中数据”“持续交易”或“待公布”，不得用未完成时段形成收盘结论。
3. 优先调用 `$futuapi` 获取指数、期货、ETF、外汇、商品和个股的收盘或最新快照，具体范围和分析方法参考后面的《必查范围》和《分析框架》。隔夜外围市场不要求获取分钟或分时K线；涉及个股新闻时可调用 `$futu-news-search` 或 `$futu-stock-digest`。
4. 使用 Web Search 系统扫描并核验分析窗口内的宏观数据、央行表态、政策监管、地缘冲突、产业链变化、重点公司公告和突发风险。优先官方机构、交易所和上市公司原始披露，其次使用高可信财经媒体；附事件发生或发布时间及链接。
5. 对可能改变当日A股预期的重要线索，比较事件发布时间与相关市场交易、收盘时间，并结合收盘快照区分“可能已定价”“尚未定价”和“仅时间上重合”。仅凭收盘快照无法确认盘中因果时，只描述相关性；事件发生在相关市场收盘后时，市场反应写“尚未验证”，不得把此前涨跌归因于该事件。
6. 若富途与网页数据冲突，比较交易时段、合约月份、币种、现货/期货、复权方式、收盘定义和时间戳；无法消除时并列披露。对不可得或口径不一致的数据明确标注缺失，不估算、不拼接。

## 必查范围

- 全球权益：道指、标普500、纳指的收盘点位与涨跌幅；标普500主要行业收盘表现；欧洲主要指数；纳斯达克金龙中国指数、中概股或其他中国相关海外资产。
- 风险定价：VIX、美元指数、美国2年期与10年期国债收益率、`2Y-10Y`期限结构、离岸人民币的收盘或最新快照。检查方向、幅度、时间戳和数据状态。
- 商品：布伦特或WTI原油、黄金、铜，以及与当日明确主题直接相关的商品。必须写清交易品种、现货或期货、合约月份和报价币种。
- 亚太盘前：富时中国A50指数期货及日经、恒指期货或其他已开盘市场，仅在品种与时间戳可验证时使用。
- 重大事件：美联储及主要央行、美国宏观数据、国内政策监管、地缘风险、贸易与制裁、重要产业链变化、AI/半导体/新能源/医药/消费等重点行业事件，以及A股或中概相关重点公司消息。
- 近期主线：将“近期”定义为目标交易日之前5个A股交易日。结合 `today.md`、`week.md` 和 `recent.md`，优先检查已有主线、候选方向、反复出现的风险和待验证观察项如何受到隔夜变量影响。

对主要权益指数、VIX、美债收益率、美元、离岸人民币和关键商品获取最近完整交易时段的收盘快照；尚未收盘或持续交易的资产使用最新快照并明确标注。比较资产时必须对齐交易日期、收盘状态和时间戳，不使用外围市场分时走势补充叙事。

不要为了“完整”填充未经核验的数据。期货、现货、ETF和指数不得混用；供应商情绪、资金流或仓位数据若无稳定公开口径，应省略或明确标注为供应商估算。

## 重大事件分级

对隔夜事件按其潜在A股影响分级：

- `一级事件`：可能直接改变指数风险偏好、全球流动性、人民币汇率、关键商品价格或A股核心行业定价，例如主要央行意外决策、宏观数据显著偏离预期、突发地缘升级或国内重磅政策。
- `二级事件`：影响特定行业、主题或重点公司，但暂不构成全市场变量，例如产业政策、龙头公司业绩或订单、海外科技链事件、商品供需扰动。
- `三级事件`：情绪或局部题材影响为主，证据不足、传导链较长或相关资产尚无确认反应。可以列入观察，但不能上升为主线判断。

分级必须同时考虑事件意外程度、影响范围、传导链长度和收盘快照中已观察到的资产反应。每个事件写明：发生或发布时间、可核验事实、预期值与实际值（如适用）、相关市场收盘状态、已观察到的市场反应、A股传导路径、支持证据、反证、待验证节点和来源链接。事件尚未被相关市场交易、或收盘快照没有对应反应时，应降低结论强度并列为待验证；不能只写标题式新闻。

## 分析框架

- `收盘结构`：比较全球权益及其行业、VIX、美债、美元、人民币和商品的收盘或最新快照，识别风险偏好、利率、汇率与商品定价是否方向一致。
- `跨资产验证`：至少使用两个逻辑相关的市场维度验证主导因子；单一资产异动只能作为线索。不同资产尚未收盘或时间戳不一致时，降低比较结论强度并说明限制。
- `证据分层`：将内容分为事实、推断和情景。事实只包括可直接核验的价格、波动、成交、公告内容、数据结果和事件时间；推断必须指出所依据的事实和反证；情景必须同时包含触发条件与失效条件。
- `定价判断`：判断隔夜主导因子属于增长、通胀、利率、美元、商品、风险偏好或重大事件冲击，并区分事件本身重要、市场已定价和市场尚未确认。没有跨资产共振时，不得强行认定唯一主导因子。
- `A股映射质量`：依次检查与近期A股主线的逻辑相关性、海外对应资产反应、人民币和A50反馈、行业代表标的共振、事件可信度及当日验证节点。对可得维度同时列出支持证据和反证；不得仅凭海外指数涨幅排名或单条消息认定A股主线。
- `上下文校验`：逐项复核 `today.md`、`week.md` 和 `recent.md` 中仍有效的主线、风险和观察项，分类为 `强化 / 削弱 / 未改变 / 待验证`，不一致时不得静默覆盖原判断。
- `盘前情景`：给出基准、偏强、偏弱三种可观察情景，不分配主观概率。每种情景说明隔夜依据、09:25或开盘后的触发条件、失效条件和主要风险。

## 输出格式

### 隔夜结论

用3至5条短句概括分析窗口内最重要的跨资产结构、主导事件、盘前风险温度及其对A股最重要的含义。风险温度使用 `偏积极 / 中性 / 偏谨慎`，并说明事实依据，不将其写成A股涨跌预测。

### 核心行情

只列可直接核验的事实。使用紧凑表格列出资产、明确品种、收盘值或最新值、涨跌幅、交易日期、时间戳和数据状态。无法取得的数据写“未取得”，不要估算。主要驱动属于推断，不放入事实表格。

### 隔夜定价结构

基于收盘或最新快照比较全球权益内部行业结构，以及权益、利率、美元、人民币、商品和中国相关资产之间的同步与背离。说明哪些资产支持主导因子、哪些构成反证，并明确尚未收盘或时间戳错位造成的限制；不描述盘中阶段、分时轨迹或有效拐点。

### 隔夜重大事件

按 `一级 / 二级 / 三级` 排序。每个事件分开说明：

- `事件事实`：发生或发布时间、事实内容、预期与实际（如适用）及来源。
- `市场反应`：相关资产收盘或最新快照的方向、幅度、时间戳，以及该事件在对应市场收盘前是否已公开；无法由快照确认因果时明确标注。
- `传导判断`：A股潜在传导路径、支持证据、反证和结论强度。
- `待验证节点`：当日可观察的确认条件与失效条件。

### A股映射与上下文复核

列出潜在受益、潜在承压和需竞价确认的风格、板块与代表标的。每项说明隔夜事实、映射逻辑、支持证据、反证、触发条件、失效条件和主要风险，不直接给出无条件买卖指令。随后使用 `强化 / 削弱 / 未改变 / 待验证` 复核运行态上下文中的已有判断。

### 今日情景与验证清单

列出基准、偏强、偏弱三种情景的隔夜依据、触发条件、失效条件和主要风险，再给出3至6个关键观察位或信号。至少覆盖09:25集合竞价、离岸人民币、A50、主要指数开盘结构、近期主线或核心标的反馈；量能条件必须使用相同市场范围、相同截止时点和相同数据源比较。

最后注明：目标A股交易日、报告生成时间、分析窗口、行情截止时间、新闻检索截止时间、数据缺口、来源链接，以及“仅供市场研究，不构成投资建议”。

## 保存并归档

最终报告生成一份Markdown文件，并保存为 `data/history/reports/${YYYY-MM-DD}/overnight-brief-${HH:mm}.md`，其中日期为报告所映射的目标A股交易日；若报告不对应A股交易日，则在文件正文中明确说明。

然后基于报告并按照 `data/context/today.md` 的规则更新 `data/context/today.md`：

1. 将仍会影响目标交易日的重大事件及其市场反应写入《盘前&盘中事件》，保留事件时间、事实、推断、反证和验证节点。
2. 更新《日内观察》前，必须同时读取 `week.md` 的本周主导变量、主线状态与周内观察，以及 `recent.md` 中仍有参考价值的近期主线、风险和验证条件。在此基础上，将隔夜报告的三种情景与这些上下文逐项交叉验证，提炼当日关键触发与失效条件并标记为 `待验证`。不得只依据本次隔夜行情新增观察项；不得删除已有观察项，重复项应合并，并记录判断相对 `week.md` 和 `recent.md` 是强化、削弱还是未改变。
3. 晨报阶段不得提前填写《日内盘面》；该部分只由实际A股交易数据续写。
