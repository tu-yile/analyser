# A股隔夜外围与重大事件晨报

## 目标

在北京时间 08:00 左右完成隔夜外围市场和重大事件扫描，提炼可能影响当日 A 股开盘、风格和行业的线索。只做基于证据的情景推演，不把外围涨跌或单一新闻机械等同于 A 股方向。

## 上下文系统

每次运行前，优先读取项目数据系统的精简运行态上下文：

- `data/context/today_state.md`
- `data/context/week_state.md`
- `data/context/recent.md`

必要时再追溯完整上下文：

- `data/context/today.md`
- `data/context/week.md`
- `data/context/recent_index.json`

报告完成后，先将最终报告保存为 Markdown 文件，再运行：

```bash
python3 scripts/market_memory.py add-report --skill overnight --report <report-file>
```

该命令会把完整报告归档到 `data/history/reports/`，并把完整报告合并到 `data/context/today.md`。

随后基于最新报告、旧 `today_state.md` 和必要的 `week_state.md` 生成新的今日状态文件，再运行：

```bash
python3 scripts/market_memory.py update-today-state --file <new-today-state-file>
```

## 工作流

1. 以 `Asia/Shanghai` 确认当前日期、星期和 A 股是否为交易日。非交易日则说明，并改为复盘最近交易时段及下一交易日风险。
2. 标注行情数据截止时间和新闻检索截止时间。美国市场未收盘、期货仍交易或关键数据尚未公布时，明确写“盘中数据”或“待公布”。
3. 优先调用 `$futuapi` 获取可用的结构化行情、K线、快照和涨跌幅；涉及个股新闻时可调用 `$futu-news-search` 或 `$futu-stock-digest`。
4. 使用 Web Search 系统扫描并核验隔夜重大事件，包括宏观数据、央行表态、政策监管、地缘冲突、产业链变化、重点公司公告和突发风险。优先官方机构、交易所、上市公司公告及高可信财经媒体，并附链接和发布时间。
5. 若富途与网页数据冲突，比较交易时段、币种、复权方式和时间戳；无法消除时并列披露，不自行选取更符合叙事的数据。

## 必查范围

- 全球权益：道指、标普 500、纳指及其行业表现；欧洲主要指数；中概股或中国相关海外资产。
- 风险定价：VIX、美元指数、美国国债收益率、离岸人民币；不可得时明确缺项。
- 商品：布伦特或 WTI 原油、黄金、铜及与当日主题直接相关的商品。
- 亚太盘前：A50 等中国资产相关期货、日经或其他已开盘市场，仅在数据可验证时使用。
- 重大事件：美联储及主要央行、美国宏观数据、国内政策监管、地缘风险、贸易与制裁、重要产业链变化、AI/半导体/新能源/医药/消费等重点行业事件、A股或中概相关重点公司消息。

不要为了“完整”填充未经核验的数据。期货、现货、ETF 和指数必须写清品种，不能混用。

## 重大事件分级

对隔夜事件按 A 股影响力分级：

- `一级事件`：可能直接改变指数风险偏好、流动性、汇率、商品价格或核心行业定价，例如美联储意外表态、重大宏观数据显著偏离、突发地缘升级、国内重磅政策。
- `二级事件`：影响特定行业、主题或重点公司，但暂不构成全市场变量，例如产业政策、龙头公司业绩或订单、海外科技链事件、商品供需扰动。
- `三级事件`：情绪或局部题材影响为主，证据不足或传导链较长。可以列入观察，但不能上升为主线判断。

每个重大事件必须写明：事件发生或发布时间、事实本身、已观察到的市场反应、A股传导路径、待验证信号。不能只写标题式新闻。

## 分析方法

先描述事实，再解释驱动，最后映射 A 股：

- 判断隔夜主导因子属于增长、通胀、利率、美元、商品、风险偏好还是重大事件冲击。
- 区分指数涨跌与内部结构，例如科技领涨、能源领涨或防御板块占优。
- 区分“事件本身重要”与“市场已经定价”。重大新闻若没有对应资产反应，应降低结论强度并列入待验证。
- 将线索映射到 A 股行业与风格，但同时给出支持条件和失效条件。
- 给出 `偏积极 / 中性 / 偏谨慎` 的盘前风险温度，并列出 2 至 4 个可在 09:25 集合竞价验证的信号。

## 输出格式

### 隔夜结论

用 3 至 5 条短句概括隔夜外围走势、最重要事件和对 A 股最重要的含义。

### 核心行情

使用紧凑表格：资产、最新值或涨跌幅、时间戳、主要驱动。无法取得的数据写“未取得”，不要估算。

### 隔夜重大事件

按 `一级 / 二级 / 三级` 排序，写明事件时间、事实、市场反应、A股传导路径、待验证信号和来源链接。

### A股映射

列出潜在受益、潜在承压和需竞价确认的方向。每项写明逻辑，不直接给出买卖指令。

### 今日验证清单

列出集合竞价、人民币、A50、量能、核心板块或核心标的需要验证的信号。

最后注明：报告生成时间、行情截止时间、数据缺口，以及“仅供市场研究，不构成投资建议”。
