# Week Context

## 规则

当前文件保存本交易周仍有分析价值的周级上下文。每次更新该文件都必须遵循以下规则。

### 各部分解释

- 《本周主导变量》
  记录本周真正取得盘面定价权的政策、宏观、产业、公司或外部变量。每项都要说明事件或预期、可能影响、实际盘面反馈以及当前有效性，不得只罗列消息，也不得用消息事后硬套行情。
- 《周内市场结构》
  归纳主要指数、同口径量能、市场广度、情绪和风格在本周的变化，重点记录结构如何演变，而不是堆叠每日收盘数据。风格至少关注大盘与小盘、价值与成长、权重与题材、红利与科技的相对强弱。
- 《主线与轮动》
  跟踪近期主线和当日辨识度板块在周内的启动、扩散、分歧、修复与失效。每个方向使用 `候选 / 已确认 / 分歧 / 已失效` 标记状态，并同时保留支持证据、反证和后续验证节点。
- 《周内观察》
  保存本周后续需要持续验证的指数位置、量能条件、市场广度、板块扩散、核心标的反馈和外部变量。观察项在周内不得直接删除，应使用 `待验证 / 已验证 / 被证伪 / 已失效` 更新状态，以保留判断演变过程。
- 《日度演变》
  按交易日期顺序追加当天对周级结构产生影响的变化，包括主导变量、市场结构、主线状态和旧观察项的验证结果。这里只写相较此前新增或改变的内容，不复制完整的 `today.md`；存在完整报告时应附相对路径。

### 如何编辑

1. 每个交易日最后一次日内复盘完成后，基于 `today.md` 和对应完整报告提炼周级增量；盘中更新只能写成带行情截止时间的阶段性结论，不得提前写成全日结论。
2. 先更新《本周主导变量》《周内市场结构》《主线与轮动》和《周内观察》，再向《日度演变》追加当日记录。
3. 只保留会影响本周后续判断的内容。单一时点的行情数字、重复事件描述和没有新增验证价值的日内噪声不进入本文件。
4. 严格区分 `事实 / 推断 / 情景`：事实必须可直接核验；推断必须写明盘面依据；情景必须同时包含触发条件和失效条件。不得把推断写成资金流事实。
5. 成交额比较必须使用相同市场范围、相同截止时间和相同数据源；市场广度、涨跌停和情绪数据口径不一致时应分别标注，不得直接拼接。
6. 主线判断不能只依据涨幅排名或单条消息。应结合指数共振、板块广度、成交支持、核心标的承接、强势持续时间和催化验证，并明确可得维度中的支持证据与反证。
7. 新分析与已有判断不一致时，不得静默覆盖旧结论；应更新原观察项或主线状态，并在《日度演变》中记录其被强化、降级、证伪或失效的原因。
8. 历史复盘只能使用行情截止时点之前已经存在的信息，并注明目标日期和时间范围，避免与当前实时市场状态混淆。
9. 每周最后一个交易日完成周终总结后，将本文件的关键内容提炼到 `recent.md`。随后只清空各内容栏目，保留本规则和组成结构供下一交易周继续使用。
